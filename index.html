(SGAE)Auto-Encoding Scene Graphs for Image Captioning

本文是CVPR2019 的关于图像描述的文章，主要让我关注的原因是用到了scene graph 和 GCN，这也是本文最大的创新点。但是本文利用的是saptial GCN（悄悄说，构建的graph节点数量少，而且是异质的，不如说是多方面融合信息已达到丰富信息的目的 :-) ）

  • 后记
    这里讲一下图卷积中spatial gcn①②③ 与 spectral gcn ④⑤⑥
    ①Learning task-dependent distributed representations by backpropagation through structure.
    ②A new model for learning in graph domains
    ③The graph neural network model
    ④Spectral networks and locally connected networks on graphs.
    ⑤Deep convolutional networks on graph-structured data.
    ⑥Semi-supervised classification with graph convolutional networks

关于这篇论文采用的graph convolutional network：采用的数

  • 为什么这样说？
    本文提到了两类graph，一类是sentence scene graph，另一类是image scene graph，而在这两类下，又进行细分为relationship、attribute、object graph。但是，每个graph 中节点是异质的，比如在relationship graph中，$obejct_a$， $obejct_b$，$relationship_ab$构成了一个graph，目的是来更新relationship embedding。从我的角度来看，只是融合相关信息来更新某一目标的特征，与graph无关。
  • 写在最前面，我个人的理解，在得到graph 节点的embedding 之后，就要输入gcn layer 来更新特征，这里，gcn layer 的表达公式可以这样理解： 该graph中所有的节点vi，经过concatenate之后，再经过一个全连接层。
    gcn(.) = fc( concatenate(v1, v2, … , vn) )**
